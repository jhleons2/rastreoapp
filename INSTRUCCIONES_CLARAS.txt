═══════════════════════════════════════════════════════════════
    🔍 VERIFICAR QUÉ VARIABLES TIENE RAILWAY
═══════════════════════════════════════════════════════════════

PASO 1: Probar el endpoint /debug
───────────────────────────────────

Abre en tu navegador:
https://rastreoapp-production.up.railway.app/debug

Esto te mostrará si las variables están configuradas.

PASO 2: Verificar en Railway Dashboard
─────────────────────────────────────────

1. Railway Dashboard → rastreoapp → Variables
2. ¿Qué variables ves?

   - Si ves PGHOST, PGPORT, etc. → Variables están compartidas
   - Si NO ves esas variables → Necesitas compartirlas

PASO 3: Si NO ves las variables
──────────────────────────────────

Tienes 2 opciones:

OPCIÓN A: Agregar Variable Reference (Automático)
1. PostgreSQL → Variables
2. Busca el link "Variable Reference"
3. Click en "+ Add Reference"
4. Selecciona "rastreoapp"

OPCIÓN B: Agregar Variables Manualmente
1. PostgreSQL → Variables
2. Copia valores de PGHOST, PGPORT, PGUSER, PGPASSWORD, PGDATABASE
3. rastreoapp → Variables → "+ New Variable"
4. Agrega cada una manualmente

═══════════════════════════════════════════════════════════════

PASO 4: Ver Logs en Railway
──────────────────────────────

1. Railway → rastreoapp → Logs
2. Busca la sección después de "Servidor corriendo"
3. Deberías ver algo como:

   📋 Variables disponibles:
      DATABASE_URL: No
      PGHOST: [valor] o No
      PGPORT: [valor] o No
      ...

Esto te dirá qué variables están disponibles.

═══════════════════════════════════════════════════════════════

SI TODAS DICEN "No":
──────────────────────

Railway NO está compartiendo las variables.

SOLUCIÓN:
1. PostgreSQL → Settings → Networking
2. Verifica que esté configurado como servicio compartido
3. O agrega las variables manualmente en rastreoapp
