PASO 1: Ir a Deployments
──────────────────────────

Railway Dashboard → rastreoapp → Deployments

¿QUÉ VES?
══════════

OPCIÓN A: "Building..." con spinner
──────────────────────────────────────
→ Railway está desplegando
→ ESPERA 1-2 minutos
→ Luego verás "Active" o "Failed"

OPCIÓN B: "Active" con checkmark verde
───────────────────────────────────────
→ Deployment completo ✅
→ Los endpoints deberían funcionar
→ Prueba los endpoints

OPCIÓN C: "Failed" con X roja
───────────────────────────────
→ Hubo error
→ Ve a "Logs" para ver el error
→ Copia el mensaje de error

═══════════════════════════════════════════════════════════════

PASO 2: Ver Logs
─────────────────

Railway → rastreoapp → Logs

¿QUÉ DEBERÍAS VER?
═══════════════════

✅ SI TODO FUNCIONA:
   "✅ Modelos sincronizados (User, Device, Location)"

❌ SI HAY ERROR:
   "Error: ..."
   "Module not found: ..."
   "Cannot find module: ..."

═══════════════════════════════════════════════════════════════

PASO 3: Probar Un Endpoint Simple
────────────────────────────────────

ABRE EN NAVEGADOR:
https://rastreoapp-production.up.railway.app/test

SI DICE:
  "Test route funciona" = servidor funcionando ✅
  "Not Found" = endpoints no están desplegados ⏳
  "Cannot GET /test" = espera un poco más

═══════════════════════════════════════════════════════════════

⏰ TIEMPO DE ESPERA
═══════════════════

Desde que haces git push hasta que Railway despliega:
→ Normalmente: 1-3 minutos
→ Si hay muchos cambios: 3-5 minutos
